============================================================
SECURITY NOTES TO INCLUDE IN README
============================================================

External LLM Risk:
External LLM providers may receive sensitive portfolio, trade, or compliance data if prompts are sent outside the organization.

Mitigation:
- Prefer local LLM using Ollama.
- Send only tool outputs required for the answer.
- Apply RBAC before tool calls.
- Never expose raw data to unauthorized roles.
- Log every interaction.
- Validate all inputs.
- Use environment variables for secrets.
- Use production-grade secrets management in real deployment.

============================================================
PRODUCTION SCALING NOTES TO INCLUDE IN README
============================================================

For production:
- Replace SQLite with PostgreSQL.
- Replace local token login with OAuth or SSO.
- Move Ollama to a dedicated inference service.
- Add API gateway.
- Add rate limiting.
- Add Prometheus and Grafana.
- Add centralized logs using ELK/OpenSearch.
- Add Kubernetes deployment.
- Add CI/CD pipeline.
- Add encryption at rest and in transit.
- Add human review workflows for high-risk trades.
